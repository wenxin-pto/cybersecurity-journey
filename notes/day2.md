# Day 2 – Linux & Defensive Security

## 🐧 Linux Fundamentals

* Linux uses a **shell (CLI)** to interpret commands and interact with the OS
* The **filesystem hierarchy (FHS)** organizes how data is stored
* The **kernel** manages processes, memory, and communication with hardware
* Package managers (e.g. APT, YUM) are used to install and manage software

## 💻 System Components

* CPU → handles computation
* RAM → temporary memory
* Hard drive → long-term storage
* Peripheral devices → external hardware (keyboard, monitor, etc.)

## 📦 Linux Distributions

* Different versions of Linux provide different tools
* Common distributions:

  * Debian-based (Ubuntu, Kali Linux)
  * Red Hat-based (CentOS)
* Kali Linux is widely used for cybersecurity testing

## 🛠️ Security Tools (Kali Linux)

* Metasploit → exploit vulnerabilities
* Burp Suite → test web applications
* John the Ripper → password cracking
* Wireshark / tcpdump → network traffic analysis

## 🛡️ Defensive Security (Blue Team)

### Key Responsibilities

* Monitoring and detecting threats
* Incident response
* Threat intelligence
* Vulnerability management
* Investigation and analysis

### Common Roles

* **SOC Analyst** → monitors systems for suspicious activity
* **Incident Responder** → investigates and responds to attacks
* **Security Engineer** → builds and maintains security systems
* **Digital Forensics Analyst** → analyzes evidence from attacks

### Defensive Measures

* Employee training
* Intrusion Detection Systems (IDS)
* Firewalls
* Security policies

---

## 📊 SIEM (Security Information and Event Management)

* Central system that collects and analyzes logs from:

  * Servers
  * Workstations
  * Security tools
* Used to detect and investigate threats

---

## 🧠 Reflection

Understanding Linux and system components helps explain how attackers interact with systems, while defensive security concepts show how organizations detect and respond to those threats.
